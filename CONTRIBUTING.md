# Contributing

Thanks for your interest in contributing to KubeVirt Observability Operator.

## Development

1. Fork the repository.
2. Create a feature branch.
3. Make your changes.
4. Run tests with:

    go test ./...
    go build ./...

5. Open a pull request.

## Pull request expectations

- Keep changes focused.
- Include tests when changing behavior.
- Update documentation when needed.
- Do not commit secrets, tokens, kubeconfigs, SSH keys, or generated binaries.
